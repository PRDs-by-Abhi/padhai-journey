Phase 1 — Notifications + Web Payment
All notification flows go live first (except Feature Paywall Abandonment), in priority order: P0 = Payment Failure, then Payment Success, then the rest. Phase 1 also includes the full web payment flow so every message can link to a working checkout.
Notifications — Deploy Order (Feature Paywall Abandonment excluded)
P01. Payment Failure Recovery
First priority. User already tried to pay. WA instant + SMS +1h + WA +24h. Links to web checkout.
2. Payment Success
On payment success trigger 2 messages: 1 WhatsApp (instant) + 1 SMS (instant). Then WA Day 3, WA Day 7. Onboard new subscribers.
3. Subscription Expiry
WA -2d + SMS -1d + WA instant + SMS +24h + WA +48h. Prevent churn.
4. Cancellation Recovery
WA +4h + SMS +24h + WA +48h. Convert checkout drop-offs.
5. Trial Activated for 7 days
One WA + one SMS instant when trial is activated on first app download. Teach users how to use the app for UPSC. Post-trial engagement handled by other flows.
6. Short-Term Win-Back
WA Day 7 + SMS Day 10 + WA Day 14. Re-engage 7–14 day inactive.
7. Weekly to Annual Upsell
3 messages within first 7 days: Day 1 (WA), Day 3 (SMS), Day 6 (WA). Optimize LTV.
8. Long-Term Win-Back
Monthly, app update, final (180+ days). Re-engage 30+ day dormant.
—Not in Phase 1: Feature Paywall Abandonment (moves to Phase 2 with product paywalls).
Infrastructure + Web Payment (Phase 1)
1WhatsApp Business API
Register and get approval for all Phase 1 utility templates (payment success/failure, expiry, cancellation, trial activated, win-back, upsell).
2TRAI DLT + SMS Gateway
Register all SMS templates (PDAIUP-S Service Implicit) with the DLT platform.
3Event Triggers + Message Queue
Backend events (payment fail, expiry, cancel, trial_activated, inactivity). Scheduler for +1h, +24h, +48h, etc. Auto-cancel on user action.
4Web Landing + Checkout
PadhAI Pro landing page, plan selection, PhonePe payment gateway only on web. Phone OTP auth.
5Payment Success / Failure on Web
Confirmation page + WA confirmation; retry page + auto-trigger Payment Failure Recovery. Capture success and failure events from PhonePe to trigger notifications.
6Subscription API + Deep Links
Activate Premium on web payment (PhonePe), sync to app. Deep link to app/store for web payers.
7Opt-out + Analytics
STOP/KEEP, DND. UTM and conversion tracking per channel.

Title
Trial Activated for 7 days — Register templates, CleverTap journey, trigger on trial activation (first app download)
Description
1) Template registration (priority 5): Register one Trial Activated template per channel (1 WA + 1 SMS = 2 templates). WhatsApp (Meta utility): 1 template, link = {link}. SMS (TRAI DLT Service Implicit): 1 template, variable = {#var#}. 2) Backend and trigger: When a user downloads the app for the first time and the 7-day trial is activated, emit trial_activated. Send user identity and event to CleverTap so this journey triggers once per user. 3) CleverTap journey: Trigger trial_activated. Steps: send WA + SMS instant (one message only). Use approved templates; respect opt-out/DND.
Acceptance
Trial Activated WA and SMS templates submitted and approved (Meta Utility, DLT Service Implicit). On trial_activated, user receives one WA + one SMS with the "we will teach you how to use the app for UPSC" messaging. No further templates in this flow; post-trial engagement uses other flows.
